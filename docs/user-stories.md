# User Stories

A task management app for neurodivergent users that eliminates decision paralysis through structured inbox processing and context-based task organization.

## Inbox/Idea Capture

**Effortless Capture:**
- As a user, I want to capture an idea in under 5 seconds without any required fields or decisions, so that I don't lose the thought or break my current focus
- As a user, I want multiple ways to get ideas into my inbox (voice note, quick text, photo of a physical note), so that I can capture them regardless of my current situation
- As a user, I want the system to accept incomplete or unclear ideas, so that I don't have to fully think through something before capturing it

**Minimal Friction:**
- As a user, I want the inbox entry to be the default screen when I open the app, so that capturing is always one tap away
- As a user, I want to capture ideas even when offline, so that lack of connectivity doesn't block my brain dump

## Inbox Processing

**Regular Processing Sessions:**
- As a user, I want to be reminded to process my inbox regularly, so that ideas don't pile up and become overwhelming
- As a user, I want to process inbox items one at a time with clear options (do now, make task, make project, delete), so that I can systematically clear my mental load

**Two-Minute Rule:**
- As a user, I want to quickly identify which inbox items can be done immediately (< 2-3 minutes), so that I can just do them and get them off my mind
- As a user, I want to complete quick actions directly from the inbox without creating formal tasks, so that simple things don't get over-systematized

**Transform to Actionable Items:**
- As a user, I want to easily convert an inbox item into a properly contextualized task when it's more than a few minutes of work, so that it enters my action system appropriately
- As a user, I want to convert vague inbox items into projects when I realize they need multiple steps, so that complex work gets properly broken down
- As a user, I want to delete inbox items that are no longer relevant, so that my system stays clean and focused

**Processing Guidance:**
- As a user, I want the system to guide me through the "what is this?" and "what's the next action?" questions during processing, so that I don't get stuck in analysis paralysis

**Inbox Visibility:**
- As a user, I want to see how long ideas have been sitting in my inbox, so that I can prioritize processing older items

## Core Task Management

**Task Processing:**
- As a user, I want to review unprocessed tasks and decide if they're single-step tasks or multi-step projects, so that I can organize my work appropriately
- As a user, I want to assign contexts to my tasks, so that I can work on things when I'm in the right environment or mindset

**Task Execution:**
- As a user, I want to see only tasks I can do right now based on my current context, so that I'm not overwhelmed by irrelevant options
- As a user, I want the system to suggest what to work on next, so that I don't get paralyzed by choice

## Project Management

**Project Creation:**
- As a user, I want to create a project when I realize something requires multiple steps, so that I can break down complex work into manageable pieces
- As a user, I want to start a project with just a description and let the system help me figure out the steps, so that I don't get stuck in planning paralysis

**Project Planning:**
- As a user, I want to add tasks to a project as I think of them, so that I can capture all the work needed without losing ideas
- As a user, I want to see all tasks within a project, so that I can understand the full scope of work

**Project Execution:**
- As a user, I want to work on project tasks alongside standalone tasks, so that I can make progress on projects without ignoring urgent single tasks
- As a user, I want to see the next actionable task from each of my projects, so that I can move projects forward without getting overwhelmed by all the steps

## Context-Based Organization

**Context Definition:**
- As a user, I want to create custom contexts based on where I am or what tools I have available, so that I can organize tasks by when they're actually doable
- As a user, I want to assign multiple contexts to a task when it could be done in different situations, so that I have more opportunities to complete it

**Context-Based Filtering:**
- As a user, I want to set my current context and see only tasks I can do right now, so that I'm not distracted by tasks I can't complete
- As a user, I want to quickly switch between contexts throughout my day, so that I can adapt to changing circumstances

**Status Management:**
- As a user, I want to mark tasks as "Waiting" and specify what I'm waiting for, so that I can track dependencies without cluttering my action lists
- As a user, I want to mark tasks as "Delegated" and track who's responsible, so that I can follow up appropriately without losing track

## AI-Assisted Features

**Project Step Generation:**
- As a user who gets overwhelmed by planning, I want the AI to suggest steps for my project based on its description, so that I can get started without getting stuck in analysis paralysis
- As a user, I want to refine AI-generated project steps by adding, removing, or modifying them, so that the plan fits my specific situation

**Task Suggestions:**
- As a user, I want the AI to suggest what task to work on next based on my context, energy level, and priorities, so that I don't waste time deciding
- As a user, I want the AI to learn my patterns and preferences over time, so that its suggestions become more helpful

**Smart Context Assignment:**
- As a user, I want the AI to suggest appropriate contexts for my tasks based on their descriptions, so that I don't have to think through every categorization

## Bullet Journal Integration

**Daily Planning:**
- As a bullet journal user, I want to generate a daily task list filtered by context and priority, so that I can write focused daily logs without decision fatigue
- As a user, I want to export today's tasks in a format that's easy to copy into my bullet journal, so that I can maintain my analog planning workflow

**Review and Reflection:**
- As a user, I want to mark tasks as completed in the app when I finish them in my bullet journal, so that both systems stay in sync
- As a user, I want to capture new tasks from my bullet journal back into the app, so that nothing gets lost between my digital and analog systems
